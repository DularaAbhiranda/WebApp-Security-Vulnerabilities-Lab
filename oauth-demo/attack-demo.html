OAuth 2.0 Vulnerability Demonstration

Vulnerable Implementation (Attack Scenario)

1. Attacker initiates OAuth flow and captures the authorization code

2. Attacker uses the code in a different session:

http://localhost/oauth-demo/vulnerable/oauth_client.php?code=stolen_code_123

3. Result: Authentication succeeds because there's no state validation

Secure Implementation (Attack Prevention)

1. Attacker attempts to use a stolen code:

http://localhost/oauth-demo/secure/oauth_client_secure.php?code=stolen_code_456

2. Result: Authentication fails with error:

"Invalid state parameter - possible CSRF attack detected"
